Pilar 01 de 07
Arquitetura de URLs
A base de tudo. URLs mal configuradas desperdiçam potencial desde o início.
0 / 6
Slug do produto contém keyword principal
Crítico

O handle do produto deve conter a keyword de maior volume, sem stopwords ou IDs automáticos gerados pelo Shopify.

/products/tenis-corrida-masculino-nike
Canonical tag configurada em todas as páginas de produto
Crítico

O Shopify cria dois caminhos para o mesmo produto: /products/ e /collections/.../products/. A canonical tag deve apontar para /products/ como URL primária em todos os temas.

<link rel="canonical" href="https://loja.com/products/slug"/>
Redirects 301 configurados para URLs renomeadas
Crítico

Toda vez que um handle de produto ou coleção é alterado, um redirect 301 deve ser criado para preservar link equity acumulado.

Admin → Navigation → URL Redirects
Estrutura de coleções em nível único
Importante

Evite hierarquias aninhadas de coleções (/collections/roupas/camisetas). Prefira coleções planas com filtros por tags para evitar duplicação de conteúdo.

Parâmetros de URL de filtro bloqueados do crawl
Importante

URLs com ?sort_by=, ?page=, ?filter.p.m= e similares geram conteúdo duplicado e desperdiçam crawl budget. Adicione noindex ou bloqueie via robots.txt.

Verifique via Google Search Console → Cobertura → Excluídos
URL da homepage sem subfolder desnecessário
Bônus

Confirme que a homepage está em domínio raiz (loja.com.br) e não em loja.com.br/home ou loja.myshopify.com redirecionando corretamente para o domínio customizado.

Pilar 02 de 07
On-Page Essentials
Os sinais on-page que o Google lê primeiro. Erros aqui são deixar dinheiro na mesa.
0 / 7
Title tag único em cada página de produto e coleção
Crítico

Cada página deve ter um title tag único, com keyword principal no início e máximo de 60 caracteres. Nunca use o template padrão do Shopify sem personalização.

Keyword Principal – Nome da Marca | Diferencial
H1 único por página contendo keyword exata
Crítico

Cada página deve ter exatamente um H1. No Shopify, o H1 geralmente é o título do produto — edite para conter a keyword principal sem variação.

Audite com Screaming Frog: H1 ausente, duplicado ou genérico
Meta description com proposta de valor e CTA
Crítico

Máximo 155 caracteres. Inclua a keyword, um diferencial competitivo (frete, prazo, garantia) e um CTA que incentive o clique. Meta descriptions influenciam CTR, não ranking.

Alt text preenchido em todas as imagens de produto
Crítico

O Shopify usa o título do produto como alt text padrão — o que é genérico. Personalize o alt de cada imagem com descrição contextual incluindo keyword natural.

Admin → Produto → clique na imagem → Alt text
H2 e H3 usados corretamente na descrição
Importante

Estruture descrições de produto com H2 para seções principais (benefícios, especificações, FAQ) e H3 para subtópicos. Melhora UX e ajuda o Google a entender a hierarquia.

SEO title personalizado no admin (não automático)
Importante

No Shopify, o title tag padrão é "Nome do Produto – Loja". Acesse Admin → Produto → Seção SEO e preencha o campo "Page title" manualmente para cada produto estratégico.

Nome das imagens com keyword antes do upload
Importante

Renomeie os arquivos antes de fazer upload. "IMG_2045.jpg" não rankeia no Google Images. Use "tenis-corrida-masculino-nike-air-zoom.webp" como padrão.

Pilar 03 de 07
Schema Markup
Rich snippets que aumentam CTR sem mover ranking. Estrelas, preço e estoque direto na SERP.
0 / 6
Schema Product validado no Rich Results Test
Crítico

Temas Shopify 2.0 incluem JSON-LD de produto nativamente. Verifique se está implementado corretamente e sem erros usando a ferramenta oficial do Google.

search.google.com/test/rich-results
Schema Review / AggregateRating ativo
Crítico

Instale um app de avaliações compatível com schema (Loox, Judge.me, Yotpo). Avaliações na SERP aumentam CTR em até 34%. Verifique se o JSON-LD está sendo injetado.

Schema Organization na homepage
Crítico

Adicione JSON-LD de Organization com nome, logo, URL, sameAs (redes sociais) e contactPoint. Essencial para o Knowledge Panel e associação da marca no Google.

Schema BreadcrumbList nas páginas de coleção
Importante

Breadcrumbs no schema melhoram a navegação na SERP e distribuem PageRank entre coleções e produtos. Adicione via metafields ou editando o theme.liquid.

Schema FAQPage nos produtos com perguntas frequentes
Importante

FAQ schema expande o snippet na SERP com perguntas e respostas diretamente no resultado. Ocupa mais espaço visual e reduz bounce pré-clique.

Schema WebSite com SearchAction para busca interna
Bônus

Permite que o Google exiba a busca interna da sua loja diretamente nos resultados. Adicione via JSON-LD na homepage com a URL da busca do Shopify.

https://loja.com/search?q={'{search_term_string}'}

Pilar 04 de 07
Core Web Vitals
Fator de ranking oficial desde 2021. LCP alto derruba posição e taxa de conversão juntos.
0 / 7
LCP abaixo de 2.5s no mobile (PageSpeed Insights)
Crítico

Largest Contentful Paint é o principal sinal de performance. Meça com PageSpeed Insights em URL mobile. Score abaixo de 60 exige ação antes de qualquer SEO de conteúdo.

pagespeed.web.dev — use URL do produto mais vendido
Todas as imagens convertidas para WebP
Crítico

WebP reduz 30–60% do peso sem perda visual perceptível. Use TinyIMG, Crush.pics ou o conversor nativo do Shopify 2.0. Meta: nenhuma imagem acima de 150kb.

Apps instalados auditados — remover sem ROI
Crítico

Cada app Shopify instala scripts que impactam o tempo de carregamento. Use Admin → Online Store → Speed para ver o impacto de cada app. Remova os que não convertem diretamente.

Lazy load em imagens fora do viewport inicial
Importante

Atributo loading="lazy" nas imagens de galeria e produtos relacionados. Temas Shopify 2.0 (Dawn) já implementam isso — confirme no código-fonte que está ativo.

<img loading="lazy" src="..." alt="...">
Preload da imagem hero (LCP element)
Importante

Adicione <link rel="preload"> no head para a primeira imagem visível do produto. Reduz LCP em 0.3–0.8s de forma mensurável. Implemente via theme.liquid.

<link rel="preload" as="image" href="{{ product.featured_image | img_url }}">
Vídeos hospedados no YouTube, não diretamente
Importante

Vídeos nativos pesam 10–50x mais que um embed de YouTube. Use iframe com loading="lazy" e parâmetro ?rel=0 para evitar sugestões de concorrentes após o vídeo.

CLS abaixo de 0.1 (sem layout shift no carregamento)
Bônus

Cumulative Layout Shift. Causado por imagens sem dimensões definidas, banners injetados por apps e fontes com flash. Defina width/height explícitos em todas as imagens.

Pilar 05 de 07
Conteúdo de Produto
Descrições copiadas do fornecedor são conteúdo duplicado. Google não ranqueia o genérico.
0 / 7
Descrições únicas com mínimo 300 palavras
Crítico

Inclua keyword principal, variações semânticas (LSI) e linguagem que corresponda à intenção de compra. Nunca copie do fornecedor sem reescrever completamente.

Use GSC para identificar produtos com impressões altas e CTR baixo
Seção de FAQ por produto com perguntas long-tail
Crítico

Adicione 3–5 perguntas que correspondam a buscas reais de compradores. Capture tráfego de cauda longa, melhore tempo de página e ative FAQ schema simultaneamente.

Conteúdo checado contra duplicação via Copyscape
Crítico

Antes de publicar, verifique se o conteúdo não está duplicado em outros sites. O Google pode desindexar páginas com alto nível de conteúdo idêntico a outras fontes.

copyscape.com — verifique os top 20 produtos
Blog com conteúdo informacional linkando para coleções
Importante

Crie posts que respondam perguntas do topo de funil e linkem para suas coleções comerciais. Ex.: "Como escolher tênis de corrida" → /collections/tenis-corrida.

Tabela de especificações técnicas em produtos
Importante

Dimensões, materiais, peso, composição, país de origem. Usuários com intenção de compra buscam specs específicas. Melhora conversão e reduz taxa de devolução.

Avaliações de clientes visíveis nas páginas de produto
Importante

Avaliações adicionam conteúdo único e atualizado que o Google valoriza como sinal de E-E-A-T (experiência real). Incentive avaliações pós-compra com automação de email.

Conteúdo gerado por usuário (fotos de clientes)
Bônus

UGC (User Generated Content) adiciona imagens únicas indexáveis, aumenta confiança e reduz necessidade de produção de conteúdo. Incentive com hashtag da marca ou concurso.

Pilar 06 de 07
Links Internos
Distribui autoridade para as páginas mais valiosas. Anchor text correto é sinal de relevância.
0 / 7
Anchor text de links internos com keyword alvo
Crítico

O texto do link interno é sinal de relevância para a página destino. Nunca use "clique aqui", "veja mais" ou "acesse". Use a keyword alvo da página que receberá o link.

Nenhuma página estratégica é órfã (sem links internos)
Crítico

Páginas sem nenhum link interno apontando para elas não recebem autoridade. Audite com Screaming Frog e vincule todas as páginas de produto a pelo menos uma coleção e um post de blog.

Produtos relacionados linkando tematicamente
Importante

A seção "produtos relacionados" deve listar itens da mesma categoria temática, não os mais vendidos aleatoriamente. Configure por coleção ou use tags para controlar a sugestão.

Posts de blog linkando para coleções comerciais
Importante

Cada post informacional deve ter pelo menos um link contextual apontando para uma coleção ou produto relevante. Não crie conteúdo de blog que não distribua autoridade para páginas de conversão.

Breadcrumbs visíveis e clicáveis no layout
Importante

Breadcrumbs melhoram navegação do usuário, reduzem bounce e criam links internos automáticos de volta às coleções. Todos os temas Shopify 2.0 incluem breadcrumbs — ative-os.

Menu principal linkando para todas as coleções-chave
Importante

Links no menu principal recebem o maior peso de link equity da homepage. Certifique-se que suas coleções mais importantes estão no menu (não apenas escondidas em submenus).

Footer com links para páginas institucionais e políticas
Bônus

Links no footer para Sobre, Política de Privacidade, Trocas e Devoluções, Contato e Rastreamento são sinais de confiança (E-E-A-T) e distribuem PageRank para essas páginas.

Pilar 07 de 07
SEO Técnico
A fundação invisível. Erros técnicos bloqueiam tudo que foi otimizado nas camadas acima.
0 / 7
Sitemap XML enviado ao Google Search Console
Crítico

O Shopify gera automaticamente em /sitemap.xml. Envie via GSC → Sitemaps e monitore erros de indexação mensalmente. Certifique-se que não há URLs bloqueadas no sitemap.

sualoja.com.br/sitemap.xml
Loja verificada no Google Search Console
Crítico

Sem GSC não há visibilidade de erros de indexação, queries de busca, problemas de cobertura ou Core Web Vitals reais. Verifique propriedade via DNS ou tag HTML no tema.

Erros 404 identificados e corrigidos com 301
Crítico

Audite com Screaming Frog ou Ahrefs. Cada página 404 com backlinks perde link equity permanentemente até o redirect ser configurado. Monitore mensalmente via GSC → Cobertura.

HTTPS ativo e sem conteúdo misto (mixed content)
Crítico

Shopify fornece SSL automaticamente. Verifique se há recursos (imagens, scripts) sendo carregados via HTTP dentro do HTTPS. Isso gera alerta de segurança no browser e penaliza confiança.

robots.txt sem bloqueio acidental de conteúdo indexável
Importante

O Shopify gera robots.txt automaticamente bloqueando /admin, /cart, /checkout. Verifique se nenhuma coleção ou produto estratégico foi acidentalmente adicionado ao Disallow.

sualoja.com.br/robots.txt
Hreflang configurado para lojas multilíngue
Importante

Se a loja opera em múltiplos idiomas (ex.: PT-BR e EN), configure hreflang para indicar ao Google qual versão exibir para cada país. O app Shopify Markets faz isso automaticamente.

Google Analytics 4 e conversões configurados
Bônus

GA4 com eventos de compra, add-to-cart e checkout permite medir o impacto real do SEO em receita. Conecte via Google & YouTube channel do Shopify para configuração simplificada.
